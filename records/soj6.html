题目描述

给定一个多项式 $\displaystyle A(x) = \sum_{i=0}^n a_i x^i$ 和 $m$ 个整数 $b_1, b_2, \cdots, b_m$，请你求出 $A \left( b_1 \right), A \left( b_2 \right), \cdots, A \left( b_m \right)$ 的值。由于答案可能会很大，所以你只需要求出答案关于 $998244353$ 取模后的值。

输入格式

第一行包含两个非负整数 $n, m$ ($0 \leq n, m \leq 10^5$)，分别表示多项式的次数和整数的个数。

第二行包含 $n + 1$ 个非负整数 $a_0, a_1, \cdots, a_n$ ($0 \leq a_i < 998244353$)，依次表示多项式的系数。

第三行包含 $m$ 个非负整数 $b_1, b_2, \cdots, b_m$ ($0 \leq b_i < 998244353$)。

输出格式

输出一行，包含 $m$ 个整数，分别为 $A \left( b_1 \right), A \left( b_2 \right), \cdots, A \left( b_m \right)$ 模 $998244353$ 的值。

题解

ps: 前置技能：多项式求逆、多项式除法及取模。

这其实就是多项式多点求值的模板问题，如何解决它呢？

如果暴力解决的话，那么复杂度将是 $O \left( n m \right)$ 的，显然不可接受。

注意到我们在做 DFT 的时候，取了一组特殊的基 ($1, w, w^2, w^3, \cdots, w^{m-1}$)，于是可以在 $O \left( n \log n \right)$ 的时间内得到它们点值。当时我们使用了分治法，利用 $\left( w^i \right)^2 = \left( w_{m/2+i} \right)^2$ 的性质将问题规模减半。(虽然说后来可以使用蝴蝶优化消除递归)

那么对于一般的一组数，我们是否也能用分治法来将问题规模减半呢？

答案是可以的。

首先，这个问题包含两个变量 $n, m$ (次数和点的个数)。我们可以不妨设 $n = m - 1$。

若 $n < m - 1$，则可以在高次项补 $0$ 得到 "次数" 为 $m - 1$ 的多项式。

如果 $n > m - 1$，则注意多项式 $P(x) = \left( x - b_1 \right) \left( x - b_2 \right) \cdots \left( x- b_m \right)$，则有 $P \left( b_i \right) = 0$，$i = 1, 2, \cdots, m$。因此我们可以将 $A(x)$ 对 $P(x)$ 取模，得到一个小于 $m$ 次的多项式 $C(x)$。且对于 $i = 1, 2, \cdots, m$，有 $A(x) = C(x)$。而这部分的时间是 $O \left( n \log^2 n \right)$ 的 (求 $P(x)$ 的分治 FFT 以及多项式除法)。

因此可设 $n = m - 1$。然而，分治法需要将 $n$ 和 $m$ 都减半，才算得到一个真正的子问题 (可以回想我们 DFT 时，将多项式的奇次项和偶次项分离，把点值也奇偶分离，得到 $n, m$ 均为原问题一般的一个子问题)。

由于点值的分布并无规律可循，因此我们可以将这些点值左右分成两份：

$$ B_1 = \left\{ b_1, b_2, \cdots, b_{\left \lfloor m/2 \right \rfloor} \right\}; B_2 = \left\{ b_{\left \lfloor m/2 \right \rfloor + 1}, b_{\left \lfloor m/2 \right \rfloor + 2}, \cdots, b_m \right\} $$

那如何才能将系数减半呢？

相信你们都猜到了。因为我们刚刚说过，在一个求值问题中，如果次数大于点值个数，则可以通过一次多项式取模来使次数和点值个数相同。

因此，令 $\displaystyle P_1 (x) = \prod_{c \in B_1} (x - c); P_2(x) = \prod_{c \in B_2} (x - c)$。则 $C_1(x) = A(x) \bmod P_1(x)$ 与点的集合 $B_1$；$C_2(x) = A(x) \bmod P_2(x)$ 与点的集合 $B_2$，就分别构成原问题的规模为 $\dfrac m2$ 的子问题。

递归终止条件为，$n = 0, m = 1$。即对一个常函数进行单点求值，此时直接返回常数项即可。

最后分析一下时间复杂度。

在第一遍递归过程中，可以使用分治 FFT 求出所有的 $P_i(x)$ (若干个一次项的乘积)，这部分的时间复杂度为 $O \left( n \log^2 n \right)$。

在第二遍递归时，对于 $n$ 的规模，需要做两次 $n$ 的多项式除法，以及两次规模为 $\dfrac n2$ 的子问题。由于多项式除法的时间复杂度为 $O \left( n \log n \right)$，因此 $T(n)$ 满足 $T(n) = 2 T \left( \dfrac n2 \right) + O \left( n \log n \right) \Rightarrow T(n) = O \left( n \log^2 n \right)$。

综上，总的时间复杂度为 $O \left( n \log^2 n \right)$，常数约为普通多项式乘法的 $10$ 倍左右。

(ps: 以下内容为 2020.2.24 更新)

这个算法虽然能过，但是常数太大了，有没有更快一点的方法？

其实是有的，下面给出的多项式多点求值做法可以避免使用多项式除法及取模，只用一次多项式求逆以及若干次普通的卷积即可完成。下面就来介绍这个做法。

设 $f \left( x \right) = a_0 + a_1 x + a_2 x^2 + \cdots + a_n x^n$，考虑代入点值 $t$，得到 $f \left( t \right) = a_0 + a_1 t + a_2 t^2 + \cdots a_n t^n$。容易发现 $f \left( t \right)$ 其实是序列 $\left[ a_0, a_1, \cdots, a_n \right]$ 和 $\left[ 1, t, t^2, \cdots, t^n \right]$ 的点积。

这好像非常平凡。不过，如果你们的想象力丰富的话，可以想象成：有一个系数翻转后的多项式 $f^R \left( x \right) = a_n + a_{n-1} x + a_{n-2} x^2 + \cdots + a_1 x^{n-1} + a_0 x^n$ 和一个特殊的多项式 $g_t \left( x \right) = 1 + t x + t^2 x^2 + \cdots + t^n x^n + \cdots$，而 $f \left( t \right)$ 恰好是 $f^R \left( x \right)$ 与 $g_t \left( x \right)$ 乘积的 $x^n$ 项系数！

而且，如果将 $g_t \left( x \right)$ 看成一个无穷级数的话 (这不影响结果)，不难发现 $g_t \left( x \right) = \dfrac 1 {1 - t x}$。

也就是说，我们只需要对每个 $t = b_i$，求出 $\dfrac {f^R \left( x \right)} {1 - t x}$ 的 $x^n$ 项系数，就完成了多点求值！

那显然不能每次除，那怎么才能优化呢？

回想我们离线求逆元的过程，是不是将所有的一次式都除下来呢？对，先将所有的一次式都除下来，得到 $f_0 \left( x \right) = \dfrac {f^R \left( x \right)} {\left( 1 - b_1 x \right) \left( 1 - b_2 x \right) \cdots \left( 1 - b_m x \right)}$，这一部分可以使用分治 NTT + 一次多项式求逆完成。

接下来，考虑我们的目标，即你现在有一个多项式 $f_0 \left( x \right)$，你要求出对于每个 $i$，将所有除了 $1 - b_i x$ 的一次式与 $f_0$ 乘起来后的 $n$ 次项系数。

根据 [soj7]多项式插值 第二部分的经验，还是考虑通过分治解决。

注意到我们在分治过程中其实已经得到 $1 - b_i x$ 乘积的一个类似线段树状的结构。于是，我们在其中搜索时，如果向左子树递归，那么就将右子树的多项式乘起来；如果享有字数递归，那么就将左子树的多项式乘起来。

但是注意到我们要 $x^n$ 项系数，那么就时刻需要保持在 $\hspace{-0.444em} \pmod {x^{n+1}}$ 下计算，从而时间复杂度又会退化成 $O \left( n \log n \cdot m \right)$，还不如暴力。

别慌啊，这当然是可以优化的：注意到我们只要 $x^n$ 项系数，那么如果我们已知剩下只需要乘一个 $d$ 次多项式的时候，我们是不是可以将过低的项去掉？

对，此时只有 $n - d \sim n$ 次项系数才能影响到答案，因此，我们不是向平常取模一样截尾，而是直接去头！

即，每次递归的时候，根据两边子树的大小，对多项式进行合适的 "去头" (即移除低次项) (比答案次数还要高的项当然要去除了，这个就不提了)

于是在递归时多项式的大小就正比于子树大小了，从而时间复杂度和之前分析一样，为 $O \left( n \log^2 n \right)$，不过注意到我们避免了多项式除法，因此常数少了不少，约为普通多项式乘法的 $6$ 倍左右。

代码

(ps: 以下内容为 2020.2.24 更新)

#include <bits/stdc++.h>
#define EB emplace_back
#define lg2 std::__lg
using std::cin;
using std::cout;

typedef long long ll;
const int N = 530000, mod = 998244353, iv2 = (mod + 1) / 2, root = 31;
typedef int vec[N], *pvec;
typedef std::vector <int> vector;

inline int & reduce(int &x) {return x += x >> 31 & mod;}
inline int & neg(int &x) {return x = (!x - 1) & (mod - x);}
ll PowerMod(ll a, int n, ll c = 1) {for (; n; n >>= 1, a = a * a % mod) if (n & 1) c = c * a % mod; return c;}

namespace Poly {
	int l, n;
	vec rev, x, y;

	void NTT_init(int len) {
		if (l == len) return; n = 1 << (l = len);
		ll g = PowerMod(root, 1 << (23 - l));
		*x = 1, *rev = 0;
		for (int i = 1; i < n; ++i)
			x[i] = x[i - 1] * g % mod, rev[i] = rev[i >> 1] >> 1 | (i & 1) << (l - 1);
	}

	void DNTT(int *d, int *t) {
		int i, *j, *k, len = 1, delta = n, R;
		for (i = 0; i < n; ++i) t[rev[i]] = d[i];
		for (i = 0; i < l; ++i) {
			delta >>= 1;
			for (k = x, j = y; j < y + len; k += delta, ++j) *j = *k;
			for (j = t; j < t + n; j += len << 1)
				for (k = j; k < j + len; ++k)
					R = (ll)y[k - j] * k[len] % mod,
					k[len] = (*k - R < 0 ? *k - R + mod : *k - R),
					*k = (*k + R >= mod ? *k + R - mod : *k + R);
			len <<= 1;
		}
	}

	vec B1, B2, B3, B4, B5, B6, B7;

	void Mul(int deg, pvec a, pvec b, pvec c) {
		if (!deg) {*c = (ll)*a * *b % mod; return;}
		NTT_init(lg2(deg) + 1);
		int i; ll iv = mod - (mod - 1) / n;
		DNTT(a, c), DNTT(b, B1);
		for (i = 0; i < n; ++i) B1[i] = (ll)B1[i] * c[i] % mod;
		DNTT(B1, c), std::reverse(c + 1, c + n);
		for (i = 0; i < n; ++i) c[i] = c[i] * iv % mod;
	}

	void Inv(int deg, pvec a, pvec b) {
		int len, i; ll iv = iv2; assert(*a);
		*b = PowerMod(*a, mod - 2), b[1] = 0, *B1 = *a, B1[1] = a[1];
		for (len = 0; 1 << len < deg; ++len) {
			NTT_init(len + 2);
			memset(b + (n >> 1), 0, n << 1), DNTT(b, B2);
			memset(B1 + (n >> 1), 0, n << 1), DNTT(B1, B3);
			for (i = 0; i < n; ++i) reduce(B2[i] = B2[i] * (2ll - (ll)B2[i] * B3[i] % mod) % mod);
			DNTT(B2, B3), std::reverse(B3 + 1, B3 + n), iv = (iv >> 1) + iv2;
			for (i = 0; i < n >> 1; ++i) b[i] = B3[i] * iv % mod;
			memcpy(B1 + i, a + i, n << 1);
		}
	}

	void Mul(vector &a, vector &b, vector &ret) {
		int degA = a.size() - 1, degB = b.size() - 1;
		if (!(degA || degB)) {ret.emplace_back((ll)a[0] * b[0] % mod); return;}
		NTT_init(lg2(degA + degB) + 1);
		int i; ll iv = mod - (mod - 1) / n;
		memcpy(B1, a.data(), (degA + 1) << 2), memset(B1 + (degA + 1), 0, (n - degA - 1) << 2);
		memcpy(B2, b.data(), (degB + 1) << 2), memset(B2 + (degB + 1), 0, (n - degB - 1) << 2);
		DNTT(B1, B3), DNTT(B2, B1);
		for (i = 0; i < n; ++i) B1[i] = (ll)B1[i] * B3[i] % mod;
		DNTT(B1, B3), std::reverse(B3 + 1, B3 + n); ret.clear(), ret.reserve(degA + degB + 1);
		for (i = 0; i <= degA + degB; ++i) ret.EB(B3[i] * iv % mod);
	}
}

namespace PolyEvaluation {
	using namespace Poly;

	int cnt = 0, lc[N], rc[N];
	vec Prd_, E1, E2, E3;
	vector g[N], tmp_;

	inline int min(const int x, const int y) {return x < y ? x : y;}
	inline int max(const int x, const int y) {return x < y ? y : x;}

	int solve(int L, int R) {
		if (L + 1 == R) return L;
		int M = (L + R) / 2, id = cnt++, lp = solve(L, M), rp = solve(M, R);
		Mul(g[lp], g[rp], g[id]);
		return lc[id] = lp, rc[id] = rp, id;
	}

	void recursion(int id, int L, int R, const vector &poly) {
		if (L + 1 == R) return tmp_.EB(poly.back());
		int i, n = poly.size() - 1, M = (L + R) / 2, lp = lc[id], rp = rc[id],
			dl = min(n, g[lp].size() - 1), dr = min(n, g[rp].size() - 1);
		if (L + 2 == R) return
			tmp_.EB((poly[n] + (ll)poly[n - 1] * g[rp].back()) % mod),
			tmp_.EB((poly[n] + (ll)poly[n - 1] * g[lp].back()) % mod);

		NTT_init(lg2(n + dr) + 1);
		ll iv = mod - (mod - 1) / Poly::n;
		vector pntt(Poly::n, 0), nxt;
		memcpy(E1, poly.data(), (n + 1) << 2), DNTT(E1, pntt.data()), memset(E1, 0, (n + 1) << 2);

		memcpy(E2, g[rp].data(), (dr + 1) << 2), DNTT(E2, E3);
		for (i = 0; i < Poly::n; ++i) E3[i] = (ll)E3[i] * pntt[i] % mod;
		DNTT(E3, E2), std::reverse(E2 + 1, E2 + Poly::n);
		nxt.clear(), nxt.reserve(dl + 1);
		for (i = n - dl; i <= n; ++i) nxt.EB(E2[i] * iv % mod);
		memset(E2, 0, (n + dr + 1) << 2), recursion(lp, L, M, nxt);

		NTT_init(lg2(n + dr) + 1);
		memcpy(E2, g[lp].data(), (dl + 1) << 2), DNTT(E2, E3);
		for (i = 0; i < Poly::n; ++i) E3[i] = (ll)E3[i] * pntt[i] % mod;
		DNTT(E3, E2), std::reverse(E2 + 1, E2 + Poly::n);
		nxt.clear(), nxt.reserve(dr + 1);
		for (i = n - dr; i <= n; ++i) nxt.EB(E2[i] * iv % mod);
		memset(E2, 0, (n + dl + 1) << 2), recursion(rp, M, R, nxt);
	}

	vector emain(int n, pvec f, const vector &pts) {
		int i, id, m = pts.size(), q;
		if (!m) return vector();
		if (!n) return vector(m, *f);
		for (i = 0; i < m; ++i) g[i].clear(), g[i].EB(1), g[i].EB(neg(q = pts[i]));

		id = solve(0, cnt = m), memcpy(Prd_, g[id].data(), (m + 1) << 2);
		Inv(n + 1, Prd_, E2), memset(Prd_, 0, (m + 1) << 2);
		if (n > 0) memset(E2 + (n + 1), 0, (Poly::n - n - 1) << 2);

		std::reverse_copy(f, f + (n + 1), E1), Mul(2 * n, E1, E2, E3),
		memset(E1, 0, (n + 1) << 2), memset(E2, 0, (n + 1) << 2);

		return tmp_.clear(), tmp_.reserve(m), recursion(id, 0, m, vector(E3 + max(n - m + 1, 0), E3 + (n + 1))), tmp_;
	}
}

int n, m, id;
vec f;
vector a;

int main() {
	int i, n, m, v;
	std::ios::sync_with_stdio(false), cin.tie(NULL);
	cin >> n >> m, a.reserve(m);
	for (i = 0; i <= n; ++i) cin >> f[i];
	for (i = 0; i < m; ++i) cin >> v, a.EB(v);
	a = PolyEvaluation::emain(n, f, a);
	for (i = 0; i < m; ++i) cout << a[i] << '\n';
	return 0;
}

坑

坑1：注意在使用 __builtin_clz 求对数时，注意特判 $n = 0$ 的情况。

坑2：在各个模板混用的时候，注意不要改变输入的数组 (多项式)，以及及时将高次项清零，避免 FFT 时出现杂项。
